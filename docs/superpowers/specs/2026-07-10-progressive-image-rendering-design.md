# ImageView 渐进式图片渲染设计

## 目标

对于大图，先显示尺寸受限的预览，再无跳变地替换为完整分辨率图片；解码工作不占用主线程，不污染完整图缓存，也不允许旧打开请求覆盖新图片。

## 设计

`ViewerViewModel.open(url:)` 在确认格式后立即建立单项导航状态，并并行启动两条异步解码：预览使用 `maxPixelSize: 2,048`，完整图使用原始分辨率。两条默认解码闭包均经由 `Task.detached` 调用 `ImageDecodeService`，因此主线程只发布状态。

预览先到时，只更新 `currentImage`。元数据、持久原图和完整图缓存仍等待完整结果，防止 HUD 与编辑操作将预览尺寸视为原始尺寸。完整图到达时替换 `currentImage`、更新元数据、写入缓存并启动既有目录扫描与邻图预加载。

每一条异步结果均检查 `openGeneration` 和当前 URL；之后发起的打开请求会让旧预览和旧完整结果失效。小图的预览与完整图相同，允许直接显示但不会影响正确性。

## 测试

1. 受控加载器暂停完整图时，预览先显示且元数据未提前发布。
2. 完整图完成后替换预览并更新真实像素尺寸。
3. 在预览或完整图等待期间打开第二个文件，旧请求不覆盖第二个文件。
4. 完整套件与生产 App bundle 构建通过。
