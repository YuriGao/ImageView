# ImageView 自由裁剪交互设计

日期：2026-07-10
状态：已确认，待实现

## 目标

为 ImageView 的基础编辑补齐自由裁剪。用户进入裁剪模式后，可在当前图片显示区域内移动和调整裁剪框；确认后，裁剪作为一项普通未保存编辑应用到当前图片。

## 范围

本次只实现自由比例裁剪，不提供固定比例、旋转校正、透视变形或裁剪历史。现有保存、放弃、切图前未保存变更提示继续复用。

## 交互

1. 通过 Edit > Crop 或 Cmd+K 进入裁剪模式；没有当前图片时入口不可用。
2. 进入时以当前可见图片区域的中心 80% 创建裁剪框，并让画布保持现有缩放与平移状态。
3. 裁剪框内部拖拽移动；四边与四角拖拽改变尺寸。裁剪框始终限制在图片显示区域内，最小显示边长为 24 pt。
4. 裁剪模式显示半透明外部遮罩、清晰边框与八个拖拽手柄。画布浏览手势、缩放和平移在此模式下不接管拖拽，避免与裁剪冲突。
5. Enter 应用裁剪，Esc 取消裁剪。确认时将画布坐标的裁剪框按当前图片绘制矩形换算成原始像素坐标，并调用既有 `.crop(CGRect)` 编辑操作。
6. 应用后退出裁剪模式，刷新图片、元数据、HUD 和未保存编辑标识；取消不会修改图片或未保存状态。
7. 选图、切图、打开文件和窗口关闭前，若仍处于裁剪模式，先退出该模式；不会把尚未确认的裁剪框视为编辑。

## 架构

`ImageCanvasView` 将暴露当前图片的绘制矩形及画布矩形到像素矩形的转换函数。这些纯几何接口由单元测试覆盖，确保缩放、偏移和坐标边界不会造成越界裁剪。

`CropOverlayView` 管理裁剪框的命中测试、拖拽状态、绘制与边界限制；其输出始终是画布坐标中的 `cropRect`。`MainWindowController` 管理模式开关、菜单/键盘路由，以及将确认后的矩形转换为 `EditOperation.crop`。图像实际裁剪仍由 `ImageEditingService` 完成。

## 错误与边界

1. 当前图片不存在、图片绘制矩形为零、或转换后的像素矩形宽高小于 1 时，确认动作不修改图片并发出系统提示音。
2. 因坐标舍入导致的边界溢出将裁剪到原始图片的像素边界。
3. 编辑服务失败时，沿用现有“无法应用编辑”错误态，并退出裁剪模式以避免界面停留在不确定状态。

## 验收与测试

1. 裁剪入口在有图片时可用，无图片时不可用；菜单、Cmd+K、Enter 与 Esc 路由正确。
2. 默认裁剪框、移动、边角调整、最小尺寸与图片边界均有单元测试。
3. 在缩放和平移后，显示裁剪框转换为正确的原始像素矩形；确认后图像尺寸符合该矩形。
4. 取消裁剪不更改 `currentImage` 或 `hasUnsavedEdits`；确认裁剪会设置未保存状态，并可通过既有保存/放弃流程处理。
