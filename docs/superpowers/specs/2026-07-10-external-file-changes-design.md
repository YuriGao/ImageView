# ImageView 外部文件变更恢复设计

## 目标

当正在浏览的图片在 ImageView 外部被删除、替换或变得不可读时，显示轻量错误态，并让用户继续浏览同目录的其他图片。

## 方案

采用按需校验而不是常驻文件系统监听。窗口重新成为活跃窗口时，以及用户请求上一张、下一张或选择胶片栏项目之前，控制器请求 `ViewerViewModel` 检查当前文件。这样覆盖用户离开 Finder 修改文件后回到 ImageView 的主路径，不引入跨目录监听、生命周期管理或后台唤醒成本。

`ViewerViewModel` 保存当前已成功显示文件的资源版本（修改时间和文件大小）。检查时：

1. 文件不存在或不可读：从 `NavigationState` 移除当前项，显示“文件已在外部移除”，并加载移除位置的下一项；没有剩余图片时清空画布。
2. 版本未变：不做任何工作。
3. 版本变化：清除该 URL 的解码缓存并重新解码。成功时更新画布和元数据；失败时显示“图片已在外部修改且无法解码”，但保留目录导航和当前显示内容，用户仍可切换。

未保存编辑优先于外部刷新：有未保存编辑时不替换当前画布，只显示外部变更提示，避免静默丢弃用户的编辑结果。

## 测试

`ViewerViewModelTests` 使用注入的文件版本读取器和解码器验证：

1. 当前文件删除后自动移除并显示下一张。
2. 当前文件被替换后重新解码并刷新元数据。
3. 外部替换为损坏内容时显示错误且仍可导航。
4. 有未保存编辑时不会以外部版本覆盖当前画布。

控制器的窗口激活路径保持薄：它只调用视图模型检查，不包含文件系统判断。
