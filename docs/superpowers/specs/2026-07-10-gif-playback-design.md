# ImageView GIF 播放设计

## 目标

播放 GIF 动图，同时确保静态图片不创建计时器、不承担逐帧解码或额外重绘负担。

## 设计

`DecodedImage` 增加可选动画帧序列，每帧包含 `CGImage` 和归一化后的显示时长。`ImageDecodeService` 仅对多帧 ImageIO 来源提取帧和 GIF 延迟；静态格式维持单张 `CGImage`，不增加任何帧数组。

`ImageCanvasView` 在赋值含动画帧的图片时启动主运行循环 `Timer`，依次更新内部显示帧并请求重绘；赋值静态图片、清空图片或视图释放时立即停止定时器。计时器只在动画图片活跃期间存在，播放速度以 GIF 每帧延迟为准，缺失或过小延迟按 100ms 处理。

编辑、预览、缓存和缩略图继续使用 `DecodedImage.cgImage` 的首帧。动画帧不参与编辑保存，避免把播放状态混入文件写回。

## 测试

1. 生成双帧 GIF 后，解码器返回两帧及相应延迟。
2. 静态 PNG 解码不创建动画帧。
3. 画布设置静态图时无播放状态；设置动画图时前进帧索引；替换为静态图后停止。
