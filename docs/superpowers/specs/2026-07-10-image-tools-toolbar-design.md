# ImageView 图片工具栏设计

日期：2026-07-10
状态：已确认，待实现

## 目标

在不破坏看图优先界面的前提下，为最常用的浏览与编辑操作提供可发现、可点击的图标工具栏。工具栏复用现有控制器命令，不引入第二套编辑或导航逻辑。

## 位置与可见性

1. 工具栏位于窗口顶部、HUD 下方，水平居中，避免和标题栏及右侧信息面板重叠。
2. 未固定 HUD 时，工具栏和 HUD 同步唤起，并在 HUD 自动隐藏时淡出。
3. 固定 HUD 时，工具栏持续可见。
4. 裁剪模式中隐藏工具栏，避免覆盖裁剪框或与 Enter/Esc 确认流程竞争。

## 内容与操作

工具栏只使用 SF Symbols 图标按钮，并通过系统 tooltip 提供命名。按钮从左到右为：上一张、下一张、顺时针旋转、裁剪、水平镜像、移到废纸篓。

1. 上一张和下一张调用现有导航路径，因此继续遵守未保存编辑提示。
2. 旋转、裁剪与镜像调用现有编辑命令，因此继续更新未保存状态、标题、HUD 和元数据。
3. 移到废纸篓调用现有确认与未保存编辑提示。
4. 没有当前图片时所有按钮禁用；第一张与最后一张的导航按钮也禁用。

## 架构

新增纯 SwiftUI `ImageToolsToolbarView`，输入为一组启用状态和动作闭包；它只负责图标、tooltip、视觉分组与按钮事件。`MainWindowController` 使用 `NSHostingView` 承载它，并在导航、图片、编辑状态、HUD 可见性与裁剪模式变化时重新渲染。

控制器将工具栏视为 HUD 可见性的一部分：`showPinnedHUD()` 与 `showHUDTemporarily()` 同步显示工具栏；HUD 的隐藏完成回调也隐藏工具栏。工具栏状态计算抽为纯函数，覆盖没有图片、目录边界及裁剪状态。

## 边界与测试

1. 工具栏不会创建新的键盘快捷键，也不会修改现有菜单验证逻辑。
2. 工具栏不显示文本标签；每个图标按钮必须有 accessibility label 和 tooltip。
3. 只有现有控制器命令具有副作用；SwiftUI 视图不会直接读写文件或编辑图像。
4. 单元测试覆盖按钮启用状态、裁剪模式隐藏条件和图标标题映射；完整套件与 App bundle 构建作为回归验证。
