# ImageView HUD 自动隐藏设计

日期：2026-07-10
状态：已确认，待实现

## 目标

让未固定的 HUD 只在用户需要时短暂显示，使图片仍然是默认界面的绝对主体；固定 HUD 继续保持常驻，服务于需要持续查看状态的用户。

## 交互

1. `Pin HUD` 开启时，HUD 始终显示，不启动自动隐藏计时器。
2. `Pin HUD` 关闭时，窗口初次显示、鼠标在窗口内移动、缩放比例改变、图片切换、或错误状态更新都会唤起 HUD。
3. 最后一次唤起后 1.8 秒没有新的交互，HUD 以短淡出动画隐藏。
4. HUD 隐藏只改变可见性，不销毁内容；下一次唤起会显示最新的文件名、序号、缩放比例和未保存状态。
5. 裁剪模式、全屏、胶片栏和信息面板不改变此规则。裁剪确认后的未保存标识会在下一次 HUD 唤起时显示。

## 架构

`MainWindowController` 维护一个可取消的主线程计时器，并通过单一 `showHUDTemporarily()` 入口更新 HUD 内容、显示视图及重新安排隐藏。窗口根视图添加鼠标移动追踪区域，将移动事件传给此入口。

固定模式由现有 `AppSettings.pinsHUD` 决定。设置切换为固定模式时立即显示并取消计时器；切换为非固定模式时按同一 1.8 秒规则开始隐藏。HUD 的 SwiftUI 视图保持只负责渲染，不持有计时或窗口状态。

## 边界与测试

1. 没有当前图片时仍可短暂显示默认 HUD，但不会创建额外的异步任务。
2. 新的唤起必须取消旧计时器，避免旧任务在新交互后错误隐藏 HUD。
3. 控制器会将“是否应保持 HUD 显示”和“是否应安排隐藏”抽成纯逻辑，覆盖固定、非固定、重复唤起与设置切换。
4. 自动隐藏不会改变 `pinsHUD` 设置，也不会影响图片缩放、导航或未保存编辑状态。
