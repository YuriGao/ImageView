# ImageView macOS 图片浏览器 PRD

首次创建：2026-07-09
最近更新：2026-07-15
状态：v1 已实现，v1.1 优化实施中

## 1. 产品定位

ImageView 是一个原生 macOS 图片浏览器，目标是替代系统 Preview，成为用户日常双击图片时的默认看图工具。

产品主打快速打开、同目录连续浏览、触控板优先的顺滑缩放和平移体验。用户从 Finder 双击一张图片后，应当立刻看到当前图片，并能自然地浏览同一文件夹中的其他图片。

## 2. 目标用户

### 主要用户

1. 设计师和开发者：经常浏览截图、UI 素材、导出图、图标和网页图片。
2. 普通 Mac 用户：希望 Preview 的替代品更快、更顺、更少打扰。
3. 图包、漫画和长图用户：需要舒服地连续浏览同目录图片，尤其重视全屏和黑底体验。

### 非主要用户

v1 不以专业摄影工作流为核心。RAW 可以作为后续或尽力兼容项，但 v1 不做专业筛片、评分、调色或色彩管理工具。

## 3. 核心价值优先级

1. 打开图片和切换图片的体感明显优于 Preview。
2. 触控板缩放、平移和切图自然、稳定、像原生系统体验。
3. 常见格式能稳定打开。
4. 界面克制、漂亮，让用户愿意把它设为默认图片打开方式。

当功能取舍发生冲突时，优先保证浏览速度和交互手感，再考虑管理、编辑和更多格式支持。

## 4. 当前产品范围（v1 已实现基线）

### 必须支持

1. 从 Finder 打开单张图片，并自动浏览同目录下的支持图片。
2. 同目录图片按文件名自然排序。
3. 支持上一张和下一张浏览。
4. 触控板优先交互：
   - 双指捏合缩放。
   - 放大后平移。
   - 左右滑动切换图片。
5. 键盘交互：
   - 左右方向键切换上一张/下一张。
   - 空格切换适应窗口和实际大小或上次缩放状态。
   - Delete 将当前图片移到废纸篓。
   - Enter 或 Cmd+Ctrl+F 进入全屏。
   - Esc 退出全屏或关闭浮层。
6. 鼠标交互：
   - 滚轮缩放或滚动。
   - 放大后拖拽平移。
   - 双击切换适应窗口和实际大小或上次缩放状态。
7. 默认极简界面，工具栏和 HUD 自动隐藏。
8. 工具栏和 HUD 可固定显示。
9. 底部胶片栏默认隐藏，可按需呼出。
10. 文件操作：
    - 移到废纸篓。
    - 重命名当前文件。
    - 在 Finder 中显示。
    - 复制当前文件路径。
11. 基础编辑：
    - 旋转。
    - 裁剪。
    - 水平镜像。
    - 垂直镜像。
    - 保存编辑结果，并明确展示未保存状态。
12. v1 必须支持格式：
    - JPEG
    - PNG
    - GIF
    - TIFF
    - BMP
    - HEIC/HEIF
    - WebP
    - AVIF
    - SVG 预览
13. 多窗口浏览：启动时显示空白窗口；第一张图片复用该窗口，其后的每张图片在独立窗口中打开；关闭最后一个图片窗口时退出应用。
14. 应用外观：支持跟随系统、浅色和深色模式。
15. 设置窗口：可选择常用图片格式并一键设为由 ImageView 默认打开。
16. 最近项目：记录成功打开的图片，并从“打开最近项目”菜单重新打开。
17. 文件夹浏览模式：
    - 从空状态、文件菜单或当前图片进入所在文件夹。
    - 按文件名搜索，按格式筛选，按名称、修改时间或大小排序。
    - 支持单选、多选、双击或 Return 打开图片，并保留文件夹会话状态。
18. 文件夹批量整理：
    - 批量移到废纸篓。
    - 批量移动到其他文件夹，冲突时支持跳过或两者都保留，绝不静默覆盖。
    - 按“基础名称 + 序号”批量重命名，并在执行前预览和校验冲突。
19. 文件夹与图片视图之间支持切换、返回和前进，并尽量保留搜索、筛选、排序、选择和滚动位置。

### 尽力支持或后续版本

1. PSD 预览。
2. RAW 预览。
3. ICO 预览。
4. ICNS 预览。
5. 更多 EXIF 信息。
6. 长图和漫画浏览优化。

### v1 明确不做

1. 不把“打开文件夹”放在 Finder 双击图片之前；单图快速打开仍是主入口。
2. 不做图库、相册、目录数据库、递归导入或后台扫描无关目录。
3. 不做收藏、标签、评分、人物、时间线等图片库能力。
4. 不做专业摄影筛片、调色和图层式编辑工作流。
5. 不做复杂标注能力，例如文字、箭头、画笔、马赛克或图层式标注。
6. 不做云同步、分享服务或 AI 功能。

## 5. 核心使用流程

### 看图主流程

1. 用户在 Finder 中双击一个支持格式的图片。
2. ImageView 打开所选图片。
3. ImageView 在后台扫描该图片所在目录中的支持图片。
4. 当前图片立即显示，目录扫描不得阻塞首次渲染。
5. 当前图片前后的若干张图片在后台预加载。
6. 用户通过触控板、键盘或鼠标浏览、缩放、平移或进入全屏。
7. 用户可呼出底部胶片栏，跳转到附近图片。
8. 用户可删除、重命名、在 Finder 中显示或复制当前图片路径。
9. 用户可旋转、裁剪或镜像当前图片，并保存结果。

### 文件夹整理流程

1. 用户从空状态、文件菜单或当前图片进入文件夹浏览模式。
2. ImageView 先显示加载状态，再逐步显示可用图片和缩略图；扫描不得冻结窗口。
3. 用户按文件名搜索、按格式筛选或按名称、修改时间和大小排序。
4. 用户通过单击、Command-单击、Shift-单击或 Command-A 选择图片。
5. 用户可打开所选图片，返回时保留原文件夹会话。
6. 用户可批量移到废纸篓、移动到文件夹或重命名，并收到明确的成功、部分失败或恢复提示。

## 6. 交互设计

### 基本原则

1. 图片是主界面，控件只在需要时出现。
2. 触控板是主路径，键盘和鼠标也必须高效、可发现。
3. 状态提示轻量但明确：文件名、序号、缩放比例、保存状态和错误状态在相关场景中可见。
4. 会改动文件的操作必须可预期、可恢复。
5. 大图优先“先看到，再变清楚”：可先显示低分辨率预览，再替换为高清渲染。

### 触控板行为

1. 双指捏合以手势中心为缩放锚点。
2. 放大后平移不能抖动、跳变或意外重置。
3. 横向滑动只在图片未放大时触发切图；手势方向遵循自然浏览方向，放大后始终用于平移，避免冲突。
4. 切图动画短、克制，并可关闭。
5. 连续快速切图时不得排队堆积长动画。
6. 双击或双指轻点可在适应窗口和实际大小或上次缩放状态之间切换。

### 视觉模式

1. 默认模式：极简界面，自动隐藏工具栏/HUD，使用中性深色或自适应背景。
2. 固定控件模式：工具栏/HUD 保持可见，方便普通用户发现功能。
3. 全屏模式：黑色或近黑背景，不显示持久控件，除非用户固定显示。
4. 胶片栏模式：底部缩略图按需出现，可随时关闭。
5. 裁剪模式：画布显示自由比例裁剪框、确认和取消控件；固定比例、旋转校正和透视变形留待后续版本。

## 7. 信息架构

### 主窗口

1. 图片画布，占据窗口主体。
2. 顶部自动隐藏 HUD/工具栏。
3. 底部默认隐藏胶片栏。
4. 可选信息面板。
5. 裁剪模式。
6. 设置窗口。
7. 文件夹浏览模式。

### HUD 和工具栏内容

1. 文件名。
2. 序号，例如 12 / 86。
3. 缩放比例。
4. 上一张和下一张。
5. 旋转。
6. 裁剪。
7. 镜像。
8. 删除。
9. 更多菜单。

### 胶片栏

1. 展示同目录图片缩略图。
2. 高亮当前图片。
3. 支持点击跳转。
4. 缩略图懒加载生成。
5. 不阻塞当前图片首次显示。
6. 选中缩略图始终位于胶片栏中心；首尾图片允许使用两侧空白保持居中。

### 信息面板

信息面板可展示：

1. 文件格式。
2. 像素尺寸。
3. 文件大小。
4. 修改时间。
5. 可用的基础 EXIF 信息。

### 文件夹浏览模式

1. 标题区明确展示当前文件夹名称，完整路径通过提示或辅助操作可见。
2. 顶部提供搜索、排序、格式筛选和批量操作。
3. 内容区使用可多选的缩略图网格，并明确展示选中状态。
4. 状态区展示图片总数、筛选结果数、选择数以及批量操作进度或结果。
5. 加载、空文件夹、筛选无结果和加载失败必须是彼此独立且可恢复的状态。
6. 窄窗口下批量操作可收纳到“更多”菜单，但不得消失或被截断。

## 8. 文件操作

1. 删除只移动到废纸篓，不永久删除。
2. 删除是否需要确认由用户设置决定。
3. 重命名后同步更新窗口标题、胶片栏项目和浏览顺序。
4. 在 Finder 中显示会打开 Finder 并选中当前文件。
5. 复制路径会复制当前文件的绝对路径。
6. 如果浏览过程中图片被外部删除、修改或变得不可读，显示轻量错误态，并允许继续浏览其他图片。
7. 设置默认打开应用时，仅修改用户勾选的格式；未勾选格式保持原有默认应用。
8. 批量移动和批量重命名必须先完成全量冲突规划，再执行文件变更。
9. 批量操作不得静默覆盖目标文件；部分失败后，失败项目保持选中并可继续处理。
10. 长时间批量操作必须显示已处理数量和总数，并允许取消尚未开始的项目。

## 9. 编辑行为

1. 旋转、裁剪和镜像会创建未保存编辑状态。
2. 应用必须展示明确的未保存状态。
3. 关闭窗口或切换到其他图片前，如果当前图片有未保存编辑，需要提示保存、放弃或取消。
4. 保存失败时不得破坏原文件。
5. 可行时采用原子写入。
6. 如果某格式无法安全写回，应提供另存为或解释限制。
7. 编辑能力不得拖慢普通浏览。

## 10. 性能要求

1. 普通图片首次渲染目标：300 ms 内显示可见内容。
2. 已预加载图片切换目标：100 ms 内显示可见内容。
3. 目录扫描不得阻塞当前图片渲染。
4. 大图可先显示低分辨率预览，再替换为全分辨率结果。
5. 支持至少 1,000 张图片的同目录浏览，打开当前图时不等待完整扫描和缩略图生成。
6. 胶片栏缩略图懒加载，并在内存上限内缓存。
7. 内存缓存优先保留当前图、附近图片和可见缩略图。
8. GIF 和动图播放不得拖慢静态图片浏览。
9. 解码失败时显示错误态，并允许继续导航。
10. 文件夹搜索、筛选和排序在 1,000 张图片的会话内，输入后的界面反馈目标为 100 ms 内。
11. 文件夹缩略图只优先加载可见区域和附近项目；离开文件夹或快速滚动后取消无效任务。
12. 原图、胶片栏和文件夹缩略图都必须有可配置或可推导的内存成本上限。

## 12. 技术建议

1. 构建原生 macOS 应用。
2. 使用 Swift 作为主要开发语言。
3. 使用 AppKit 承担窗口、手势、滚动、缩放和文件集成等精细交互。
4. SwiftUI 可用于 HUD、工具栏、设置页和简单面板。
5. 以 ImageIO、CoreGraphics 和 CoreImage 作为主要解码、渲染和图像处理基础。
6. 仅在系统框架无法提供可接受的 WebP、AVIF、SVG、PSD 或 RAW 行为时，引入聚焦的第三方库。
7. 使用后台队列处理目录扫描、图片解码、缩略图生成和预加载。
8. 主线程只负责渲染和交互。
9. 文件写入采用安全写入策略，降低文件损坏风险。

## 13. 验收标准

1. 从 Finder 双击支持格式图片后，能打开当前图并识别同目录图片。
2. 当前图片可在目录扫描完成前显示。
3. 上一张/下一张可通过触控板、键盘和鼠标完成。
4. 触控板缩放和平移稳定，正常使用中不与切图冲突。
5. 控件隐藏时界面足够干净。
6. 工具栏/HUD 可固定显示。
7. 底部胶片栏可显示和隐藏。
8. 胶片栏不需要等待所有缩略图生成后才可用。
9. 删除会将当前图片移到废纸篓。
10. 重命名会更新当前文件和 UI 状态。
11. 在 Finder 中显示会选中当前文件。
12. 复制路径会将绝对路径放入剪贴板。
13. 旋转、裁剪和镜像可以保存。
14. 未保存编辑在关闭或切图前会提示。
15. 不支持或损坏的图片显示明确错误态。
16. 1,000 张图片目录可打开当前图，不等待全部缩略图。
17. v1 必须支持格式都有明确测试覆盖。
18. 基础编辑失败不得损坏原文件。
19. 第一张图片、批量打开的后续图片和最近项目均遵循多窗口规则，窗口状态彼此独立。
20. 关闭最后一个图片窗口时应用退出；设置窗口不阻止退出。
21. 胶片栏在切图和窗口尺寸变化后保持选中缩略图居中。
22. 文件夹搜索、筛选和排序可组合使用，并在返回文件夹视图后保持原状态。
23. 文件夹批量操作绝不静默覆盖文件；部分失败可继续操作，恢复提示包含具体文件和原因。
24. 键盘可完成文件夹选择、打开、全选和批量移到废纸篓；焦点位置始终可见。
25. 图标按钮、文件夹单元格、选中状态、加载和操作结果对 VoiceOver 提供可理解的名称、状态和变化通知。

## 14. 版本规划

### v1.0

高速单图打开、同目录连续浏览、触控板优先看图、胶片栏、基础文件操作、基础编辑和主流格式支持。

### v1.1

优先修复缩放语义和功能可发现性；补齐文件夹上下文、选择反馈、批量进度与无障碍基线。详细需求见第 17 节。

### v1.2

增加非破坏性编辑历史、标准撤销/重做、可停靠信息面板和更完整的批量操作恢复能力。

### v1.3+

优化长图和漫画式浏览；增强 PSD、RAW、ICO 和 ICNS 兼容；扩展 EXIF 和色彩配置显示。是否加入导出或格式转换，必须先验证不会损害默认看图工具的轻量定位。

## 15. 风险

1. WebP、AVIF、SVG、PSD 和 RAW 支持可能受 macOS 版本和解码策略影响。
2. 触控板手势在平移和切图之间容易冲突。
3. 编辑保存如果设计不谨慎，可能损坏文件或造成格式信息丢失。
4. 胶片栏生成和预加载如果缓存策略不当，会增加内存占用。
5. 原生手感依赖 AppKit 级别的精细交互，仅靠声明式 UI 可能不够精确。
6. 极简界面如果隐藏了过多能力，会造成用户只会“打开图片”，却不知道如何编辑、查看信息或进入文件夹整理。
7. “适应窗口倍率”和“实际像素倍率”如果使用同一个百分比口径，会误导需要核对尺寸的设计师和开发者。
8. 文件夹批量能力扩大了误操作影响范围，进度、取消、冲突处理和恢复信息必须先于更多批量功能。

## 16. 当前开放决策

1. 当前最低支持 macOS 14；是否下调版本必须单独评估，不作为近期优化目标。
2. WebP、AVIF、SVG、PSD 和 RAW 的兼容矩阵、降级行为和测试素材集。
3. 非破坏性编辑历史确定为最多 20 步，只保存操作参数而非位图快照；完整图与连续阅读共享 512 MiB 解码缓存预算，单张图片自身超过预算时允许仅保留当前图，不继续预取邻图。
4. 文件夹批量移到废纸篓和重命名的撤销承诺边界；无法可靠撤销时必须明确告知。
5. 缓存上限确定为：完整图与连续阅读 512 MiB、动图帧 128 MiB、胶片栏缩略图 128 MiB；文件夹缩略图只由可见单元持有并在复用时释放。发布 RSS 门槛为小图 180 MiB、8,000 × 8,000 大图 1,100 MiB、24 帧 1,200 × 1,200 动图 300 MiB、1,000 张目录 200 MiB。基准脚本为 `scripts/run-memory-benchmarks.sh`，本轮四类场景全部通过，实测见 [内存基准](../../assets/performance/memory-baseline-2026-07-15-113013.md)。
6. 是否加入匿名性能数据。默认不联网；若未来考虑遥测，必须单独征得用户同意并公开字段。

### 17.3 P0：缩放语义与输入一致性

#### 目标

让用户在任何入口看到的倍率和实际显示含义一致，避免尺寸判断错误，同时保留触控板优先的自然操作。

#### 需求

1. 明确四种显示状态：`适应窗口`、`实际大小 100%`、`手动倍率`、`上次手动倍率`。
2. 状态栏在适应窗口时显示“适应窗口”，或同时显示“适应窗口 · 82%”；不得只显示容易被理解为实际像素倍率的“100%”。
3. 实际大小定义为 1 个图片像素对应 1 个屏幕逻辑点；Retina 下是否额外提供“1 图片像素 = 1 物理像素”模式需另行评估，不混入 100% 定义。
4. 双击和 Space 使用同一状态机：在适应窗口与实际大小或上次手动倍率之间切换。
5. 菜单、状态栏、双击、Space、捏合和带修饰键滚轮必须更新同一倍率状态，不出现入口间语义漂移。
6. 输入规则固定为：触控板捏合缩放；放大后双指或鼠标拖拽平移；未放大时横向触控板手势切图；Command 或 Option + 滚轮缩放。普通鼠标滚轮在无法滚动时不得无反馈地假装执行操作。
7. 状态栏倍率支持点击，提供适应窗口、50%、100%、200% 和自定义倍率的快捷选择。

#### 验收标准

1. 使用 400 × 400、2,000 × 2,000 和超长图片测试时，状态栏计算误差不超过 1 个百分点。
2. 双击、Space、菜单“实际大小”和倍率菜单得到一致结果。
3. 连续捏合或滚轮缩放以指针或手势中心为锚点，不跳回画布中心。
4. 处于适应窗口时，用户不再看到孤立且容易误解的“100%”。

### 17.4 P0：功能可发现性与首次使用

#### 目标

在不破坏极简画布的前提下，让首次使用者发现浏览、信息、编辑和整理能力。

#### 需求

1. 标题栏保留文件夹切换按钮，并增加一个始终可见的“更多”按钮；更多菜单至少包含旋转、裁剪、显示胶片栏、显示信息、另存为、在 Finder 中显示和移到废纸篓。
2. 更多菜单复用现有系统菜单命令、图标和快捷键，不新建第二套行为。
3. 所有图标按钮提供本地化辅助名称、悬停提示和快捷键；提示在浅色、深色和高对比度下可读。
4. 第一次成功打开图片后显示一次轻量提示，说明左右切图、捏合缩放、Space 切换显示模式和更多菜单；用户一旦交互或 6 秒后自动消失，之后不重复打扰。
5. 设置中提供“重新显示使用提示”；帮助窗口改为可浏览、可搜索的快捷键与手势列表，不再只有单段说明。
6. 空状态在主操作下方展示最多 5 个本地最近项目，并提供清除入口；文件不存在时自动移除。最近项目只存本机，不上传。
7. 全屏模式仍默认隐藏非必要控件；鼠标移动或键盘操作时按现有自动隐藏节奏出现。

#### 验收标准

1. 打开图片、切换下一张、查看信息和旋转四项任务均能从可见按钮、更多菜单或 macOS 菜单到达，并在可搜索帮助中找到；命令映射、菜单可用状态和本地化文案由自动化测试覆盖。
2. 任何核心命令都能从可见按钮、更多菜单或 macOS 菜单栏到达，并能看到快捷键。
3. 使用提示不会在第二次启动重复出现，且可从设置恢复。
4. 增加入口后，未展开控件时画布可用面积不减少。

### 17.5 P0：文件夹上下文、批量反馈与响应式布局

#### 目标

让用户始终知道自己在哪个文件夹、看到多少图片、选中了什么，以及批量操作进行到哪一步。

#### 需求

1. 文件夹模式标题显示文件夹名称；悬停或辅助操作展示完整路径。图片模式继续显示当前文件名。
2. 工具栏左侧为搜索，中间为排序和格式筛选，右侧为批量操作；搜索框宽度限制在 220–420 pt，不无限占满剩余空间。
3. 状态区固定展示“共 N 张”“筛选后 M 张”“已选择 K 张”；没有选择时不显示选择数。
4. 批量操作按安全程度排序：移动、重命名、移到废纸篓；移到废纸篓保持破坏性视觉语义，不与普通筛选控件混为一组。
5. 窗口变窄时，低频批量操作按优先级收纳到更多菜单；搜索、选择数和当前操作进度必须保持可见。
6. 批量任务展示“已处理 X / Y”、当前阶段和取消按钮。取消只阻止尚未开始的项目，已完成项目不回滚并在结果中说明。
7. 完成后显示成功数、失败数和“查看详情”；失败项目保持选中。仅当系统返回可靠恢复位置时提供“撤销移到废纸篓”，批量重命名只在反向规划仍无冲突时提供撤销。
8. 从图片返回文件夹时保留搜索、筛选、排序、选择、滚动位置和最近打开项；无必要不得重新扫描。

#### 验收标准

1. 在 700 pt、900 pt 和 1,200 pt 宽窗口中，工具栏无重叠、截断或不可达操作。
2. 1,000 张图片目录中，搜索和筛选输入后 100 ms 内出现界面反馈；缩略图继续异步加载。
3. 批量处理 100 个文件时，进度单调递增，取消后未开始项目保持原状，结果数量之和等于请求数量。
4. 冲突、权限失败和源文件消失三类情况都不会覆盖目标文件，并能从结果中定位到具体项目。

### 17.6 P0：键盘、VoiceOver 与运动偏好

1. 文件夹网格项目暴露文件名、格式、选中状态和位置，例如“第 3 项，共 28 项”。
2. 搜索、排序、格式筛选、批量操作、图片/文件夹切换、信息面板和更多菜单都可通过键盘到达，焦点环清晰可见。
3. 加载完成、筛选结果变化、批量操作完成或失败通过无障碍通知播报，但连续缩略图加载不得造成重复噪声。
4. 应用设置中的切图动画开关继续保留；系统开启“减少动态效果”时必须覆盖应用动画设置并关闭非必要过渡。
5. 使用系统语义颜色和控件状态，在浅色、深色、增加对比度和降低透明度设置下验证可读性。

### 17.7 P1：非破坏性编辑历史

1. 每次旋转、镜像和裁剪形成一条编辑操作，原始解码结果保持不变，显示结果由操作序列派生。
2. `Cmd-Z` 撤销一步，`Shift-Cmd-Z` 重做一步；“放弃编辑”恢复到打开时状态并清空历史。
3. 标题栏继续使用已编辑标记，同时在更多菜单或图像菜单中显示可撤销操作名称，例如“撤销裁剪”。
4. 历史默认最多 20 步，并受内存成本限制；大图优先保存操作参数而不是完整位图副本。
5. 保存采用现有安全写入策略。保存成功后当前结果成为新基线；保存失败时保留历史和可见结果。
6. 切图、关闭窗口、外部文件变化与保存冲突继续复用现有保存、放弃、取消保护。

### 17.8 P1：信息面板与元数据操作

1. 信息面板支持“临时浮层”和“停靠侧栏”两种形态；临时查看使用浮层，持续对比时停靠并重新计算画布可用区域，避免直接遮住图片。
2. 增加完整路径、颜色空间、色彩配置、位深、方向和可用 EXIF；缺失字段不显示空行。
3. 文件名、路径、像素尺寸和拍摄时间可单独复制；提供“在 Finder 中显示”。
4. 面板内容可滚动，窄窗口下不越界；关闭按钮、固定按钮和复制操作均提供键盘与 VoiceOver 支持。

### 17.9 P2：长图和漫画阅读

1. 新增“适应宽度”模式，允许纵向滚动长图；缩放和平移不再与切图手势冲突。
2. 可选连续纵向阅读同目录图片，默认仍为单图模式，不改变 Finder 双击入口。
3. 连续模式只预取当前图附近 2–3 张，并遵守统一内存预算。
4. 支持从左到右或从右到左的上一张/下一张方向设置，但不引入书架、收藏或阅读数据库。

### 17.10 成功指标与验证方法

1. 核心任务可达性：打开、切图、查看信息、旋转均具备确定性的可见入口、菜单入口或快捷键，且首次提示与可搜索帮助覆盖对应操作；不以招募外部用户或主观计时作为发布阻塞条件。
2. 冷打开可见内容：标准测试集 p50 不高于 300 ms，p95 不高于 800 ms；已预加载切图 p95 不高于 100 ms。
3. 文件安全：自动化故障注入下静默覆盖和原文件损坏为 0；所有部分失败都有逐文件结果。
4. 稳定性：每个版本发布前，现有自动化测试全部通过，并增加缩放语义、无障碍属性、批量取消和编辑历史测试。
5. 内存：建立小图、大图、动图、1,000 张目录四类基准；具体上限在 P0 性能测量后写回第 16 节，不以未经测量的固定数字作为承诺。
6. 隐私：以上指标通过本地基准、测试脚本和维护者本机检查验证，不依赖联网遥测或外部用户访谈。

### 17.11 实施顺序

1. P0-A：缩放语义、状态栏倍率和统一输入状态机。
2. P0-B：标题栏更多菜单、首次提示、帮助内容与最近项目入口。
3. P0-C：文件夹标题、计数、响应式工具栏、批量进度和取消。
4. P0-D：键盘、VoiceOver、减少动态效果和高对比度验证。
5. P1-A：非破坏性编辑历史与标准撤销/重做。
6. P1-B：可停靠信息面板和元数据操作。
7. P2：长图、漫画阅读和后续格式兼容。

所有阶段都必须保留“快速单图打开、同目录连续浏览、界面克制、文件操作安全”四项核心价值。若优化与首次渲染或切图性能冲突，先保证核心浏览性能，再缩减附加表现。
